# Roadmap and project history

This page points at two canonical documents: **what was built** (original plan) and **what might come next** (V2 backlog). It does not duplicate the tables—open the sources for the full text.

## Original plan reference

[**ORIGINAL_PLAN_REFERENCE.md**](https://github.com/OWNER/REPO/blob/main/docs/ORIGINAL_PLAN_REFERENCE.md) is the **historical** implementation ledger: workstream IDs (e.g. docker-compose, warehouse views, webhooks, secrets) with completion checkmarks. Use it to answer **“why is this in the tree?”** and to trace v1 **scope** without reading old chat logs.

## V2 backlog

[**V2_BACKLOG.md**](https://github.com/OWNER/REPO/blob/main/docs/V2_BACKLOG.md) is the **living** roadmap after v1: status workflow (`proposed` → `done`), goals (reliability, large libraries, security, multi-tenant), and **numbered** backlog items (e.g. `v2-rel-001`).

Contributors should **append** new ideas there when discovery surfaces follow-ups, per project process.
